# Submission Checklist

## Before Applying

- Choose the correct resume version:
  - Startups role: `11_resume_drafts.md` Resume Version A.
  - Codex role: `11_resume_drafts.md` Resume Version B.
- Use the matching cover letter from `06_cover_letters.md`.
- Link the proof-pack README as the main portfolio entrypoint.
- Link only cleaned projects from `03_project_index.md`.
- Do not link rough or unrelated repos.

## Application Form Language

Use this positioning sentence:

> I am a non-traditional AI deployment candidate focused on practical LLM systems: multi-agent orchestration, AI-assisted coding workflows, human-in-the-loop pipelines, eval-driven reliability, and India-specific startup use cases. My audit background gives me strong instincts for traceability, failure modes, documentation, and operational controls.

## Do Not Say

- My profile is weaker than the formal checklist.
- I can learn anything.
- I am passionate about AI.
- I have many unfinished projects.
- Any casual language that makes the work sound disposable.

## Say Instead

My path is non-traditional, but my portfolio shows the work: building AI workflows, designing agent systems, writing specs, creating evals, adding reliability controls, and using AI coding tools in real development loops.

## After Applying

- Record application date in `13_application_tracker.md`.
- Send 15-25 targeted OpenAI/APAC outreach messages.
- Send 5-10 Indian startup feedback messages.
- Send 3-5 Codex or AI coding workflow feedback messages.
- Publish or share one public post draft within 48 hours of applying.
